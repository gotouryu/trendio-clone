# AI台本生成プロンプト検討メモ 2026-05-22

## 前提

- Gemini APIキー未設定時はMockでデモ動作する。
- Gemini APIキー設定後は `/api/ai-content` が `GEMINI_MODEL` に指定したモデルで実生成する。
- 参考URLは解析しない。入力条件の一部として、構成や雰囲気の参考扱いにする。

## 入力項目の扱い

### 必須にする項目

| 項目 | 理由 |
|---|---|
| ターゲット | 誰の悩み・欲求に刺すかが決まらないと企画が一般論になる |
| 投稿テーマ | 何を見せる動画かが決まらないと台本が撮影指示まで落ちない |

### 入力を強く推奨する項目

| 項目 | 理由 |
|---|---|
| 絶対に入れたい内容 | 訴求ポイント、強み、根拠、NG回避条件をAIへ固定できる |
| CTA | 動画の締めと企画のゴールがぶれにくい |
| 動画の長さ | リールのシーン数と秒数を決める |
| 演者 / ナレーション | 撮影可能な台本へ寄せる |

### 任意でよい項目

| 項目 | 理由 |
|---|---|
| 参考URL | 現仕様ではURL内容を解析しないため、URLだけで品質は上がらない |
| 会社名 | ブランド名を自然に出したい時に有効。企画骨子には必須ではない |
| 会社URL | 現仕様ではWebサイト内容を読まないため、URLだけで品質は上がらない |

## 追加候補の実装

2026-05-22 に入力画面へ追加:

1. 投稿目的: 認知、保存、問い合わせ、購入、来店、採用応募
2. 訴求ポイント / 根拠: 価格、特徴、実績、比較優位、提供エリアなど
3. 避けたい表現: 誇大表現、競合比較、使いたくない語句など
4. 動画トーン: 親しみ、信頼、驚き、上質、専門家風、テンポ重視
5. 使える素材: 演者、店舗、商品、画面録画、口コミ画像、制作前後素材など

プロンプトにも流し込み、企画の方向、映像指示、CTA導線、避ける表現の制御に使う。

## 企画生成プロンプト候補

### 候補A: 切り口分散型

ターゲット、投稿テーマ、必須要素、CTAを必ず反映しつつ、3案を「課題共感型」「意外性・検証型」「信頼・実績型」のように明確に分散させる。広告っぽい押し売りを避け、冒頭2秒のフックと視聴後行動まで設計させる。

評価:
- 強み: 似た企画が並びにくく、デモで「選べる感」が出る。
- 弱み: 商材情報が薄い場合はやや一般論になりやすい。

### 候補B: バズ優先型

保存、コメント、シェアを狙うため、違和感、比較、チェックリスト、ランキング、Before/Afterなどの型を優先して企画化する。フックの強さを最優先する。

評価:
- 強み: 冒頭の引きが強くなりやすい。
- 弱み: BtoBや堅い業種では煽りが強くなりすぎる可能性がある。

### 候補C: 顧客課題逆算型

ターゲットが抱える不安、面倒、損失、憧れを先に言語化し、その感情から企画を作る。各案に「視聴者の悩み」「動画で得られること」「CTAへの自然な接続」を含める。

評価:
- 強み: お客様が求める内容に寄りやすく、業種が変わっても安定する。
- 弱み: 出力項目が増えるため、画面表示用には少し圧縮が必要。

### 候補D: ブランド安全型

過度な断定、薬機法・景表法リスク、誇大表現を避け、信頼感を重視して企画化する。医療、美容、金融、教育などに強い。

評価:
- 強み: 炎上・審査リスクが低い。
- 弱み: デモでの派手さは弱い。

## 企画生成で採用する2案

1. 候補A: 切り口分散型
2. 候補C: 顧客課題逆算型

理由:
- デモでは複数案の違いが見えることが重要。
- 実務では、お客様の入力に対して「それっぽい一般論」ではなく、ターゲット課題に寄った案が必要。
- 現在の実装プロンプトはAをベースに、Cの考え方を追加できる構造にしている。

## 台本生成プロンプト候補

### 候補A: 撮影指示具体化型

各シーンに、秒数、映像、ナレーション、テロップ、SEを必ず出す。映像は「何を撮るか」が分かる表現に限定し、抽象語だけを禁止する。

評価:
- 強み: 実際に撮影・編集へ移しやすい。
- 弱み: 出力が長くなりやすい。

### 候補B: 冒頭維持率重視型

冒頭2秒で結論または違和感を置き、3〜8秒ごとに画変わりを作る。最後はCTAに自然接続する。尺の合計一致を厳守する。

評価:
- 強み: リール向けの構成になりやすい。
- 弱み: 落ち着いたブランドではテンポが速すぎる可能性がある。

### 候補C: セールス導線型

認知、興味、理解、信頼、行動の順にシーンを作る。CTAの説得力を上げるため、途中に根拠や実績を入れる。

評価:
- 強み: 問い合わせや購入に向いた台本になりやすい。
- 弱み: SNSらしい軽さは弱くなる場合がある。

### 候補D: ナレーションなし対応型

ナレーションなしの場合、captionだけで意味が通るように設計する。演者なしの場合は、商品、画面録画、資料、手元、店舗、テロップ中心で成立させる。

評価:
- 強み: 入力条件への追従性が高い。
- 弱み: 単独だと企画の尖りは弱い。

## 台本生成で採用する2案

1. 候補A: 撮影指示具体化型
2. 候補B: 冒頭維持率重視型

理由:
- デモ画面で一番価値が伝わるのは「すぐ撮れる粒度」の台本。
- Instagramリール用途では冒頭2秒と短尺の画変わりが重要。
- Dの条件追従は、現在の実装プロンプト内の必須条件として吸収済み。

## 自己テスト

### テスト条件

- ターゲット: 20代の美容好きな女性
- 投稿テーマ: 水分不足チェックリストのInstagramリール
- 演者: あり
- ナレーション: なし
- 絶対に入れたい内容: 水を飲む重要性、生活習慣チェック、最後にLINE登録CTA
- 動画の長さ: 30秒
- CTA: 公式LINEに登録
- 会社名: 株式会社Karteia

### 判定

- 企画生成は、A単独よりA+Cの組み合わせが最も安定。企画の差分とターゲット課題の両方が出る。
- 台本生成は、A+Bの組み合わせが最もデモ向き。シーン別に「映像・テロップ・SE・秒数」が出て、短尺動画として成立しやすい。
- ナレーションなし条件は、プロンプトの必須条件で明示しないと narration に文章が入りやすいため、現在の実装では `narration` を空文字に正規化する保険も入れている。
